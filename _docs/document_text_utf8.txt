1. Introduction
1.1 Product Overview
LeadMap Pro is a cloud-based SaaS web application that enables businesses, sales teams, marketing agencies, and entrepreneurs to discover, qualify, and export high-intent B2B leads by intelligently querying real-world data from Google Maps and supplementary public web sources.
Unlike existing tools that are either too technical (raw scrapers) or too limited (browser extensions), LeadMap Pro is designed to be the definitive no-code lead intelligence platform â€” combining Google Maps data extraction, AI-powered enrichment, competitor mapping, and CRM-ready export into a single, beautiful, real-time dashboard.
1.2 Vision Statement
PRODUCT VISION
"To be the world's most intuitive lead intelligence platform â€” turning Google Maps' 200 million+ business listings into a real-time, filterable, AI-enriched lead database that any business can use in under 60 seconds, with no technical knowledge required."
1.3 Problem Statement
The core problem this product solves is threefold:
Finding qualified leads is manual and time-consuming. Sales teams spend hours copying business data from Google Maps row by row.
Existing scraping tools are either too technical (requiring code and proxy management) or too limited (capped at 120 results, no enrichment).
There is no single tool that combines lead discovery, competitor mapping, review intelligence, and CRM export in one clean interface.
MARKET VALIDATION
The web scraping industry crossed $1.03 billion in 2025 and is projected to reach $2 billion by 2030 (14.2% YoY growth â€” Mordor Intelligence). Google Maps contains 200M+ business listings across 220+ countries, with 2 billion monthly active users. 88% of consumers use Google Maps to find local businesses (BrightLocal, 2026). The Google Places API costs $17â€“$35 per 1,000 requests for bulk data â€” making a dedicated SaaS solution 10x more cost-effective at scale.
1.4 Objectives
Deliver a no-code lead generation tool accessible to non-technical users within 60 seconds of signup.

Provide AI-powered competitor analysis, gap detection, and market saturation scoring.

3. Legal & Compliance Framework
3.1 Legal Basis for Data Collection
Scraping publicly available Google Maps data is a legally nuanced area. The following framework governs LeadMap Pro's approach:
KEY LEGAL PRECEDENT
In hiQ Labs v. LinkedIn (2019, affirmed 2022), the US Ninth Circuit ruled that scraping publicly visible data (data accessible without logging in) does not violate the Computer Fraud and Abuse Act (CFAA). This ruling is the primary legal basis for commercially operating Google Maps scraping tools. Google's Terms of Service prohibit scraping, but ToS violations are a private contract matter (risk: account ban), not a criminal or civil law matter.

3.3 Data Handling Policy
All extracted business data is treated as commercial/factual data, not personal data, under the GDPR distinction.
Individual email addresses collected via website enrichment are tagged as 'enriched contact' and subject to opt-out rights.
Scraped data is stored in encrypted databases with a configurable data retention period (default: 90 days).
No scraped data is sold to third parties. Data belongs to the user who initiated the scrape.
All scraping operations are rate-limited server-side to avoid causing server harm to Google's infrastructure.
Users must agree to a Data Use Policy before exporting lead lists, confirming compliant outreach use.

4.3 Scraping Engine â€” Technical Deep Dive
4.3.1 The 120-Result Cap Problem
Google Maps only displays a maximum of 60â€“120 business listings per search query, regardless of how many businesses exist in an area. This is the primary technical challenge all Google Maps tools must solve.
GRID SEARCH SOLUTION
LeadMap Pro's Grid Search Engine divides any target geographic area into a dynamic grid of smaller sub-areas (cells). Each cell is small enough that it contains fewer than 60 businesses, meaning a full extraction is possible from each cell. The cells are then merged, deduplicated by Place ID, and returned as a unified result set.
Algorithm: (1) User defines target area. (2) System queries the area with an initial broad search. (3) If result count approaches 60, the area is recursively split into 4 quadrants. (4) Each quadrant is queried independently. (5) Results are merged and deduplicated using Google Place IDs as unique keys.
4.3.2 Anti-Detection Strategy
Residential Proxy Rotation: Every request is routed through a different residential IP address via Bright Data or Oxylabs pools. Residential IPs mimic real user traffic.
Request Rate Limiting: Server enforces minimum 2â€“5 second delays between requests per proxy, with randomized jitter. Configurable per user tier.
Browser Fingerprint Randomization: Playwright launches with randomized viewport, user agent, language headers, and timezone to avoid fingerprint-based detection.
Session Rotation: New browser contexts are created per scrape job, not per request, to mimic natural user sessions.
CAPTCHA Handling: Integration with 2Captcha or Anti-Captcha API for automated CAPTCHA solving when encountered.
Retry Logic: Exponential backoff with a maximum of 3 retries per failed request before job failure notification.
4.3.3 Data Enrichment Pipeline
After extracting base Google Maps data, the enrichment pipeline runs asynchronously for each lead with a website URL:
Website crawler visits the business website using Scrapy with a rotating user agent.
Email Extraction: Parses the homepage, /contact, /about pages for email addresses using regex + NLP.
Social Profile Detection: Identifies LinkedIn, Facebook, Instagram, Twitter/X links in page HTML.
Tech Stack Fingerprinting: Reads HTTP headers, HTML meta tags, and script sources to identify CMS, analytics, and advertising pixels (e.g. WordPress, Shopify, Google Ads pixel).
Contact Page Crawl: If no email found on homepage, visits the /contact or /contact-us path specifically.
All enrichment data is stored asynchronously and pushed to the user's results panel via WebSocket in real time.

9.2 Credit System Logic
1 credit = 1 extracted lead from Google Maps (base data). Enrichment does not cost additional credits in V1.
Credits are consumed when a lead is successfully extracted and returned to the user's results panel.
If a search returns 0 results due to a scraping error, no credits are charged.
Credits roll over for 30 days from the billing date (not calendar month) to avoid end-of-month rush.
Users can purchase top-up credit packs at $5 per 500 credits without upgrading their plan.
